# Contributing

## Feature Requests

tiny_window_manager is not accepting any new feature requests at this time, sorry. You can file a feature request for a feature that you plan to implement and submit a pull request for, so that the feature can be reviewed and you will know ahead of time if the feature will be rejected.  

## Bugs

Please search through the existing issues, open and closed, before filing a new bug.
Add the version of tiny_window_manager, the version of the OS, and screenshots or videos as necessary.

## Coding Style

Please match the existing coding style as much as possible.

## License

By contributing to tiny_window_manager you agree that your contributions will be licensed under its MIT license.

## Incentives

Logic from tiny_window_manager is used in the [Multitouch](https://multitouch.app) app. The [tiny_window_manager Pro](https://tiny_window_managerapp.com/pro) app is entirely built on top of tiny_window_manager. If you contribute significant code or localizations that get merged into tiny_window_manager, you get a free license of Multitouch or tiny_window_manager Pro. Contributors to Sparkle, MASShortcut, or Spectacle can also receive free Multitouch or tiny_window_manager Pro licenses (just send me a direct message on [Gitter](https://gitter.im)).
